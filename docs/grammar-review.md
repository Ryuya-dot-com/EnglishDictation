# 文法の復習

このページは、`questions.json` にある文法カテゴリを、日本の英語学習者向けに短く復習できるよう整理したものです。  
解説は、文法用語を並べるだけでなく、`何を表す形か` `どう見分けるか` `ディクテーションでどこを落としやすいか` に重点を置いています。
カテゴリ別の 4 択確認は [../grammar-quiz.html](../grammar-quiz.html) で解けます。問題ソースは [grammar-review-quiz.md](grammar-review-quiz.md) にあります。

## 読み方

- `ひとことで言うと`: その文法が何を表すか
- `形の目印`: 何を見つければよいか
- `コアイメージ`: 訳語より先に押さえたい感覚
- `よくあるつまずき`: 混同しやすい点
- `ディクテーションでの着眼点`: 聞き取りと語順再現のポイント
- `入試で問われやすいポイント`: 空所補充、整序、書き換えで狙われやすい箇所
- `頻出誤答例`: 実際に崩れやすい形と、その直し方
- `比較メモ`: 似た文法と最短で見分けるための一言
- `例文`: `questions.json` からの代表例
- `関連カテゴリ`: 近い項目、混同しやすい項目

## 受験での使い方

- まず `形の目印` だけを読んで、見分ける合図を覚えます。
- 次に `よくあるつまずき` を読んで、自分が混同しやすい項目を確認します。
- そのあと `例文` を音読し、必要なら `関連カテゴリ` に移ります。
- 空所補充では、`動詞の形`, `to / -ing`, `that / who / where` のような短い語を先に疑うと得点しやすいです。
- 並べ替えでは、先に `文の骨格`, 次に `節`, 最後に `修飾` を並べると崩れにくいです。

## 特に混同しやすい組み合わせ

- `条件文` と `仮定法過去 / 仮定法過去完了`
  - 条件文は実際に起こりうる話、仮定法は現実とずれた話です。
- `間接疑問` と `疑問文`
  - 間接疑問は平叙文語順です。
- `名詞節` と `関係代名詞`
  - 名詞節は節全体が文の部品になり、関係代名詞は前の名詞を説明します。
- `形式主語` と `It to構文` と `It that構文` と `形式目的語`
  - `It` が本当の意味を持つかではなく、後ろに本体があるかを見ます。
- `現在分詞` と `過去分詞` と `分詞構文`
  - 名詞説明か、文全体説明かを先に確認します。
- `結果構文` と `目的構文` と `too〜to構文` と `enough構文`
  - 程度の結果なのか、意図なのか、可能か不可能かを見分けます。

## 頻出の見分け方早見表

- `if + 現在形, will + 動詞の原形`
  - 条件文の基本です。
- `if + 過去形, would / could + 動詞の原形`
  - 仮定法過去です。
- `if + had + 過去分詞, would have + 過去分詞`
  - 仮定法過去完了です。
- `名詞 + who / which / that + 節`
  - 関係代名詞の可能性が高いです。
- `疑問詞 + 主語 + 動詞`
  - 間接疑問の語順です。
- `what節 / that節 / whether節` が主語や目的語に入る
  - 名詞節です。
- `It is + 形容詞 + to do`
  - It to構文です。
- `It is + 評価語 + that ...`
  - It that構文です。
- `find / make / consider + it + 形容詞 + to do / that節`
  - 形式目的語です。
- `have / has + 過去分詞`
  - 現在完了をまず疑います。
- `had + 過去分詞` に、基準になる別の過去がある
  - 過去完了の可能性が高いです。
- `名詞 + -ing`
  - 現在分詞の後置修飾を疑います。
- `名詞 + 過去分詞`
  - 過去分詞の後置修飾を疑います。
- `文頭の -ing / Having + 過去分詞 + ,`
  - 分詞構文の可能性が高いです。

## 頻出誤答パターン

- `if it will rain`
  - 条件文なら `if it rains` が基本です。`if節` にふつう `will` は置きません。
- `I don't know where did he go.`
  - 間接疑問なので `I don't know where he went.` のように平叙文語順に戻します。
- `the book what I bought`
  - `the book that / which I bought` か、先行詞を含む `what I bought` かのどちらかです。
- `I am used to get up early.`
  - `be used to` の `to` は前置詞なので、後ろは `getting` になります。
- `He suggested that she goes home.`
  - 仮定法現在なら `goes` ではなく原形の `go` です。
- `It is difficult understand this problem.`
  - `It is + 形容詞 + to do` なので、`to understand` が必要です。
- `This is the place which I was born.`
  - 場所を表すなら `where I was born`、または `which I was born in` です。
- `He made me to clean the room.`
  - 使役動詞 `make` の後ろは `to` なしの原形で、`made me clean` です。
- `It was so interesting movie that I read it twice.`
  - `so + 形容詞` ではなく、名詞を伴うなら `such an interesting movie that ...` が自然です。
- `He has gone there yesterday.`
  - `yesterday` のような明確な過去時点があるなら、現在完了より `went` が基本です。
- `If I were rich, I will travel around the world.`
  - 仮定法過去なので、主節は `would travel` にします。
- `The man talked to me yesterday is my teacher.`
  - 名詞を後ろから説明するなら `The man who talked to me yesterday ...` のように関係詞が必要です。

## 迷ったときの判定順

1. まず `主語` と `動詞` を探します。
2. `if`, `that`, `who`, `where`, `to`, `-ing`, `had`, `have` のような合図を拾います。
3. 文の中心が `節` なのか `句` なのかを判断します。
4. 次に、`時制`, `助動詞`, `語順` のどこが問われているかを見ます。
5. 最後に、似た文法との違いを 1 つだけ確認します。

この順番で見ると、長文中の一文でも、空所補充でも、整序英作文でも判断がぶれにくくなります。

## さらに迷いにくくする共通ルール

- `条件文 / 仮定法`
  - 未来のことでも、ふつうの `if節` は現在形にします。`if節` が過去形なら、まず「現実との距離」を表す仮定法を疑います。
- `間接疑問`
  - `yes / no` 型は `if / whether`、`wh-` 型は `疑問詞 + 主語 + 動詞` の平叙文語順です。`whether to do`, `whether or not`, `前置詞 + whether` は特に狙われやすい形です。
- `関係詞`
  - 前に名詞があり、後ろの節で主語や目的語などが足りないなら関係詞を考えます。定義用法ではコンマを打たず、目的格の関係代名詞は省略されることがあります。
- `不定詞 / 動名詞`
  - 後ろの形は「前の動詞が決める」と考えると整理しやすいです。`remember / forget / try` は `to do` と `doing` で意味が変わるので、かたまりで覚えます。
- `現在完了`
  - 「今まで」や「今に残る影響」があるなら現在完了を疑います。`yesterday` や `last week` のように終わった過去時点があれば、まず過去形を考えます。
- `受動態`
  - 形は `be + 過去分詞`、助動詞があるなら `助動詞 + be + 過去分詞` です。誰がしたかより、何がどうされたかを前に出す形だと考えると見分けやすいです。

## 重要比較まとめ

### 1. 名詞節 / 間接疑問 / 関係代名詞 / 関係副詞

**まず見ること**

- 前に説明される名詞があるか
- 節全体が主語・目的語・補語に入っているか
- 疑問の意味を保っているか

**見分け方**

- `名詞節`
  - 節全体が文の部品です。
  - 例: `What he said surprised me.`
- `間接疑問`
  - 疑問の内容を埋め込んでいます。
  - 語順は平叙文です。
  - 例: `I don't know where he lives.`
- `関係代名詞`
  - 前の名詞を後ろから説明します。
  - 例: `The boy who lives next door is kind.`
- `関係副詞`
  - 前の名詞の「場所・時・理由・方法」を説明します。
  - 例: `This is the place where we met.`

**最短で区別する質問**

- 前に名詞がない:
  - 名詞節か間接疑問の可能性が高いです。
- 前に名詞がある:
  - 関係詞の可能性が高いです。
- 疑問の意味が残っている:
  - 間接疑問をまず疑います。
- `what` が単独で使われている:
  - 先行詞を含む名詞節の可能性が高いです。

**受験で効く一言**

- `that` は「つなぐだけ」になることが多いです。
- `what` は先行詞を自分で含みます。
- `where / when / why` は、前に `place / time / reason` 系の語があるかを見ます。

### 2. 条件文 / 仮定法過去 / 仮定法過去完了

**まず見ること**

- `if節` の時制
- 主節に `will`, `would`, `would have` があるか
- 現実に起こりうる話か、反事実か

**見分け方**

- `条件文`
  - `if + 現在形`, `主節 + will ...`
  - 現実に起こりうる話です。
- `仮定法過去`
  - `if + 過去形`, `主節 + would / could ...`
  - 今の事実と違う想像です。
- `仮定法過去完了`
  - `if + had + 過去分詞`, `主節 + would have + 過去分詞`
  - 過去の事実と違う想像です。

**最短で区別する質問**

- `if節` が現在形:
  - まず条件文を疑います。
- `if節` が過去形:
  - 仮定法過去の可能性があります。
- `if節` が `had + 過去分詞`:
  - 仮定法過去完了を強く疑います。
- 主節が `will`:
  - 条件文寄りです。
- 主節が `would have`:
  - 仮定法過去完了です。

**受験で効く一言**

- `if it will rain` は基本形ではありません。
- `If I were you` は丸ごと押さえるべき定型です。
- `would have` が見えたら、まず反事実を考えます。

### 3. 形式主語 / It to構文 / It that構文 / 形式目的語

**まず見ること**

- `It` が文頭か、動詞の後ろか
- 後ろに `to do` が来るか、`that節` が来るか
- `It` が本当に何かを指しているか

**見分け方**

- `形式主語`
  - `It seems that ...` のように、後ろの内容が本体です。
- `It to構文`
  - `It is important to ...` のように、`to do` が本体です。
- `It that構文`
  - `It is true that ...` のように、`that節` が本体です。
- `形式目的語`
  - `find it easy to ...`, `make it clear that ...` のように、`it` の後ろに本体があります。

**最短で区別する質問**

- `It` のあとにすぐ動詞:
  - 形式主語の可能性があります。
- `It is + 形容詞 + to do`:
  - It to構文です。
- `It is + 評価語 + that ...`:
  - It that構文です。
- `動詞 + it + 形容詞 + to do / that節`:
  - 形式目的語です。

**受験で効く一言**

- `It` に意味を載せすぎないことが大切です。
- 本当の内容は、たいてい後ろの `to do` か `that節` にあります。
- 並べ替えでは、`it` の位置を先に固定すると崩れにくくなります。

### 4. 現在分詞 / 過去分詞 / 分詞構文

**まず見ること**

- その `-ing / 過去分詞` が、名詞を説明しているか、文全体を説明しているか
- 名詞が「する側」か「される側」か
- コンマがあるか

**見分け方**

- `現在分詞`
  - 名詞を後ろから説明し、能動の感じを出します。
  - 例: `the boy playing soccer`
- `過去分詞`
  - 名詞を後ろから説明し、受け身や完了の感じを出します。
  - 例: `the homework given by the teacher`
- `分詞構文`
  - 文全体に「時・理由・条件」などを添えます。
  - 例: `Walking along the river, ...`

**最短で区別する質問**

- 直前に名詞がある:
  - 現在分詞か過去分詞の修飾を疑います。
- コンマで区切られている:
  - 分詞構文の可能性が高いです。
- 名詞が動作をする側:
  - 現在分詞寄りです。
- 名詞が動作を受ける側:
  - 過去分詞寄りです。

**受験で効く一言**

- `the girl singing ...` と `the letter written ...` の違いを、意味で説明できるようにします。
- 分詞構文は、主節との主語一致まで確認すると誤りを防ぎやすいです。

### 5. 現在完了 / 過去完了 / 完了不定詞

**まず見ること**

- `have / has / had` のどれか
- 基準になる時点が今か、過去か
- `to have + 過去分詞` になっていないか

**見分け方**

- `現在完了`
  - `have / has + 過去分詞`
  - 今につながる意味があります。
- `過去完了`
  - `had + 過去分詞`
  - 過去のある時点より前を表します。
- `完了不定詞`
  - `to have + 過去分詞`
  - 主節より前の内容を不定詞で表します。

**最短で区別する質問**

- 基準が今:
  - 現在完了の可能性が高いです。
- 基準が過去:
  - 過去完了を疑います。
- `to have` が見える:
  - 完了不定詞です。

**受験で効く一言**

- `yesterday` があれば現在完了は普通使いません。
- 過去完了は「過去の中のさらに前」です。
- 完了不定詞は、主節との時間差を見ると判定しやすくなります。

## 1. 文の骨格とつながり

### 接続詞

**ひとことで言うと**  
接続詞は、文と文をつないで、時間・理由・条件・逆接などの関係を示す語です。

**形の目印**  
`when`, `because`, `if`, `although`, `as soon as` などの後ろに、ふつう `主語 + 動詞` が続きます。

**コアイメージ**  
全部を日本語に置き換えるより、前後の文の流れをどうつないでいるかを先に取ると理解しやすくなります。`because` なら理由、`when` なら時、`although` なら逆らう流れです。

**よくあるつまずき**  
前置詞句と混同しやすい点に注意します。`because of` の後ろは名詞ですが、`because` の後ろは節です。

**ディクテーションでの着眼点**  
接続詞が聞こえた瞬間に、後ろの節の役割を予測します。これができると、長い文でも切れ目を見失いにくくなります。

**入試で問われやすいポイント**  
接続詞は、意味選択と節のつながりがよく問われます。`because / although / when / if` の違いを日本語訳ではなく前後の論理で見分けられるかが得点差になります。

**頻出誤答例**  
`Because of it was raining, we stayed home.` は不自然です。節を続けるなら `Because it was raining ...`、名詞を置くなら `Because of the rain ...` にします。

**比較メモ**  
接続詞は節と節をつなぎます。後ろが名詞なら前置詞句を疑い、後ろが `主語 + 動詞` なら接続詞を考えます。

**例文**

- `As soon as I got home, I started my homework.` 「家に帰るとすぐに宿題を始めました。」

**関連カテゴリ**

- 条件文
- 譲歩表現
- 名詞節

### 条件文

**ひとことで言うと**  
条件文は、「もし〜なら」「〜する限り」のように、ある条件とその結果を結びつける形です。

**形の目印**  
`if`, `unless`, `as long as`, `provided that` がよく使われます。基本の条件文では、未来のことでも `if節` は現在形になることが多いです。

**コアイメージ**  
大事なのは、「条件を言う部分」と「結果を言う部分」の2つがあることです。意味の軸は `if節` よりも、文全体の因果関係にあります。

**よくあるつまずき**  
`if it will rain` のように `if節` に `will` を入れたくなりやすいですが、普通の条件文では `if it rains` になります。仮定法との違いも意識が必要です。

**ディクテーションでの着眼点**  
`if` や `unless` が聞こえたら、後ろに条件節、その後に主節が来ると予測できます。時制の取り違えを防ぐのに役立ちます。

**入試で問われやすいポイント**  
`if節` の時制と `主節` の助動詞の組み合わせは、空所補充と整序英作文で非常によく問われます。`if it will rain` のような誤りを避け、普通の条件文なのか仮定法なのかを見分けることが得点に直結します。

**頻出誤答例**  
`If it will be sunny tomorrow, we will play tennis.` ではなく、普通の条件文なら `If it is sunny tomorrow, we will play tennis.` が基本です。

**比較メモ**  
条件文は「起こりうる話」が基本です。`if節` が現在形ならまず条件文、`if節` が過去形なら仮定法を疑います。

**例文**

- `If it rains tomorrow, the game will be canceled.` 「明日雨が降れば、試合は中止されます。」

**関連カテゴリ**

- 接続詞
- 仮定法過去
- 仮定法過去完了

### 結果構文

**ひとことで言うと**  
結果構文は、「とても〜なので、その結果...」という流れを表す形です。

**形の目印**  
`so + 形容詞 / 副詞 + that ...`、`such + 名詞 + that ...` が基本です。

**コアイメージ**  
前半で程度を強く出し、後半でその結果を述べます。単に「とても〜」ではなく、「そのせいでこうなった」という結果までがセットです。

**よくあるつまずき**  
`so that` は目的構文にも出るので注意が必要です。結果構文では前半に `so + 形容詞 / 副詞` があり、目的構文では「〜できるように」の意味になります。

**ディクテーションでの着眼点**  
`so` が聞こえたら、後ろに程度表現が来るのか、目的の `so that` なのかを早めに判定します。`such` が聞こえたら、その直後は名詞が来やすいです。

**入試で問われやすいポイント**  
`so ... that ...` と `such ... that ...` の使い分けは頻出です。`so` の後ろは形容詞・副詞、`such` の後ろは名詞中心になることを押さえると失点しにくくなります。

**頻出誤答例**  
`He is so a kind boy that everyone likes him.` は誤りです。名詞 `boy` を伴うので `He is such a kind boy that ...` にします。

**比較メモ**  
結果構文は「その結果こうなる」型です。`so ... that` で程度が先なら結果、`so that S can ...` で意図が強ければ目的構文を考えます。

**例文**

- `She is so kind that everyone likes her.` 「彼女はとても親切なので、みんなに好かれています。」

**関連カテゴリ**

- 目的構文
- 比較
- 接続詞

### 目的構文

**ひとことで言うと**  
目的構文は、「〜するために」「〜できるように」という目的を表す形です。

**形の目印**  
`to do`, `in order to do`, `so that + 主語 + can / could ...` が代表的です。

**コアイメージ**  
結果ではなく、話し手や主語の意図を言う形です。まだ起きていない未来向きの内容が来やすいのが特徴です。

**よくあるつまずき**  
`so that` は結果構文と混同しやすいですが、目的構文では「〜できるように」の気持ちが入ります。`to do` は不定詞の副詞的用法としても出てきます。

**ディクテーションでの着眼点**  
`in order to` や `so that` が聞こえたら、後ろに来る動詞は「何のためか」を説明しています。主文の動作とのつながりを意識すると再現しやすいです。

**入試で問われやすいポイント**  
`to do`, `in order to do`, `so that S can ...` の言い換えは非常によく問われます。結果構文との違いは「実際に起きた結果」ではなく「そうする意図」があるかで見分けます。

**頻出誤答例**  
`He studied hard for pass the exam.` は不自然です。目的なら `He studied hard to pass the exam.` または `so that he could pass the exam` の形にします。

**比較メモ**  
目的構文は「そうするために」という意図を表します。実際に起きた結果を述べるなら結果構文、単に後ろから名詞を説明するなら不定詞の形容詞的用法です。

**例文**

- `He is studying hard so that he can pass the exam.` 「彼は試験に合格できるように一生懸命勉強しています。」

**関連カテゴリ**

- 結果構文
- 不定詞
- 接続詞

### 譲歩表現

**ひとことで言うと**  
譲歩表現は、「〜であっても」「たとえ〜でも」と、前の条件に逆らう内容を表す形です。

**形の目印**  
`although`, `even if`, `no matter what`, `however`, `whoever` などが合図になります。

**コアイメージ**  
「それでも」という逆らう感じが中心です。前半で不利な条件を出しても、後半で結論は変わらないことを示します。

**よくあるつまずき**  
条件文と似ていますが、条件文は結果が変わる可能性を示し、譲歩表現は条件があっても結論が変わらないことを示します。

**ディクテーションでの着眼点**  
`no matter` や `even if` が聞こえたら、その後に何が来ても主節の結論はぶれないと予測できます。長い文でも意味の軸を保ちやすくなります。

**入試で問われやすいポイント**  
`although`, `even if`, `no matter wh-` の選択と意味の違いは狙われやすいです。条件文と違って「条件があっても結論が変わらない」と判断できるかが重要です。

**頻出誤答例**  
`Although he was tired, but he kept studying.` は二重の逆接です。`Although he was tired, he kept studying.` または `He was tired, but he kept studying.` にします。

**比較メモ**  
譲歩は「それでも」を表します。条件しだいで結論が変わるなら条件文、条件があっても結論が変わらないなら譲歩表現です。

**例文**

- `No matter what happens, I will support you.` 「何が起こっても、私はあなたを支えます。」

**関連カテゴリ**

- 接続詞
- 条件文
- 相関接続詞

### 相関接続詞

**ひとことで言うと**  
相関接続詞は、2つの語や句をペアで結びつける表現です。

**形の目印**  
`both A and B`, `either A or B`, `neither A nor B`, `not only A but also B`, `whether A or B` などが基本です。

**コアイメージ**  
前半を聞いた時点で、後半の形もある程度予測できるのが特徴です。意味より先に「ペアで働く形」だとつかむと強いです。

**よくあるつまずき**  
前半と後半の品詞や形をそろえる必要があります。`either A or B` では動詞の一致が後ろの主語に引かれる場合もあります。

**ディクテーションでの着眼点**  
`not only` が出たら `but also`、`either` が出たら `or` を待てるようにします。ペアの後半を予測できると聞き漏らしが減ります。

**入試で問われやすいポイント**  
ペアの後半を正しく補う問題や、前後の品詞をそろえる問題が出やすいです。`either A or B` で動詞がどちらに一致するかも頻出です。

**頻出誤答例**  
`Not only he can play the piano but also sing.` は語順が崩れています。倒置するなら `Not only can he play the piano, but he can also sing.` です。

**比較メモ**  
相関接続詞は前半を見た時点で後半が決まるペア表現です。単独の接続詞と違い、前後の品詞や形をそろえる視点が必要です。

**例文**

- `He not only plays the piano but also sings well.` 「彼はピアノを弾くだけでなく、歌も上手です。」

**関連カテゴリ**

- 接続詞
- 譲歩表現
- 比較

### 名詞節

**ひとことで言うと**  
名詞節は、節なのに文の中で名詞のような働きをする部分です。

**形の目印**  
`that節`, `what節`, `whether節`, `if節` などが、主語・目的語・補語の位置に入ります。

**コアイメージ**  
一つの節全体で「こと」「もの」「〜かどうか」のような内容を表します。文のどこに入っているかを見ると整理しやすいです。

**よくあるつまずき**  
関係代名詞や間接疑問と形が似ています。名詞節は、節全体が文の部品になっている点が重要です。

**ディクテーションでの着眼点**  
`what`, `that`, `whether` が聞こえたら、その後ろが一まとまりで主語や目的語になる可能性を考えます。節の終わりまで切らずに聞くことが大切です。

**入試で問われやすいポイント**  
`that / what / whether / if` の選択は頻出です。節全体が主語・目的語・補語のどこに入っているかを見れば、関係詞や副詞節との区別がしやすくなります。

**頻出誤答例**  
`I know that what he said.` は不自然です。`what he said` がすでに名詞節なので、`I know what he said.` とします。

**比較メモ**  
名詞節は節全体が主語・目的語・補語になります。前に先行詞があれば関係詞、疑問の意味が強ければ間接疑問を先に疑います。

**例文**

- `What is important is that you keep trying.` 「重要なのは挑戦し続けることです。」

**関連カテゴリ**

- 間接疑問
- 関係代名詞
- 形式主語

### 間接疑問

**ひとことで言うと**  
間接疑問は、「何を」「どこで」「いつ」などの疑問内容を、文の一部として埋め込んだ形です。

**形の目印**  
`I know where ...`, `Please tell me when ...`, `Can you explain how ...` のように、疑問詞の後ろが平叙文語順になります。`〜かどうか` なら `if / whether + 主語 + 動詞` になります。

**コアイメージ**  
疑問文そのものではなく、「〜かを知っている」「〜かを教えて」の中身として疑問を入れています。だから語順は疑問文ではなく平叙文になります。

**よくあるつまずき**  
`Where did he go?` の語順をそのまま埋め込みたくなりますが、`I know where he went.` のように `主語 + 動詞` の順に戻します。`whether` は `whether or not`, `whether to do`, `前置詞 + whether` の形で特に使われやすい点も重要です。

**ディクテーションでの着眼点**  
疑問詞が聞こえても、すぐに疑問文だと決めないことが大事です。前に `know`, `tell me`, `wonder` などがあれば、間接疑問の可能性が高いです。

**入試で問われやすいポイント**  
最大の頻出ポイントは語順です。`where he went` のように平叙文語順に戻せるかどうか、また「〜かどうか」の `if / whether` を条件の `if` と区別できるかがよく問われます。`whether or not`、`whether to do`、`前置詞 + whether` の形も定番です。

**頻出誤答例**  
`I don't know where did he go.` は誤りです。間接疑問では `I don't know where he went.` のように平叙文語順に戻します。

**比較メモ**  
疑問の意味が残っていて、後ろが平叙文語順なら間接疑問です。前に先行詞があるなら、まず関係詞との違いを確認します。

**例文**

- `Please tell me where you bought this bag.` 「このバッグをどこで買ったのか教えてください。」

**関連カテゴリ**

- 名詞節
- 疑問詞＋to不定詞
- 接続詞

### 関係代名詞

**ひとことで言うと**  
関係代名詞は、前の名詞を後ろから詳しく説明するための語です。

**形の目印**  
`who`, `which`, `that` の後ろに節が続き、その節が直前の名詞を説明します。後ろの節では、主語か目的語などの要素が足りないことが多いです。

**コアイメージ**  
「どの人か」「どの物か」を後ろから限定する働きです。関係代名詞そのものが、後ろの節の中で主語や目的語の役割も持ちます。

**よくあるつまずき**  
接続詞の `that` と見分けにくいですが、関係代名詞なら前に必ず説明される名詞があります。`what` は先行詞を含むので別扱いです。定義用法ではコンマを打たず、関係代名詞ですでに役割を果たしているので後ろで代名詞を重ねません。

**ディクテーションでの着眼点**  
名詞の直後に `who / which / that` が来たら、「ここから後ろは説明」と判断します。名詞と節の切れ目を意識すると語順が崩れにくくなります。

**入試で問われやすいポイント**  
先行詞の種類と、後ろの節の中で足りない要素が主語なのか目的語なのかを見抜く問題が多いです。特に `that` と `what` の区別、目的格関係代名詞の省略、後ろに余分な代名詞を置かないことは定番です。

**頻出誤答例**  
`The book what I bought yesterday is interesting.` は誤りです。先行詞 `the book` があるので、`what` ではなく `that / which` を使います。

**比較メモ**  
前に先行詞があり、後ろの節で主語か目的語が足りないなら関係代名詞です。前に先行詞がなければ `what節` や名詞節を疑います。

**例文**

- `The book that I borrowed from the library was interesting.` 「私が図書館から借りた本は面白かったです。」

**関連カテゴリ**

- 関係副詞
- 名詞節
- 接続詞

### 関係副詞

**ひとことで言うと**  
関係副詞は、場所・時・理由・方法を表す名詞を後ろから補足する語です。

**形の目印**  
`where`, `when`, `why`, `the way` がよく出ます。前に `place`, `day`, `reason`, `way` などの先行詞があることが多いです。

**コアイメージ**  
関係代名詞が「人・物」を説明するのに対して、関係副詞は「場所・時・理由・方法」を補います。後ろの節は、その名詞の中身を具体化します。

**よくあるつまずき**  
`the way` は特別で、学校文法では `the way (that) S V` の形をよく扱います。`the way how` は普通の書き方ではあまり使いません。`where` や `when` の後ろは節としてはほぼ完成していることが多いので、関係代名詞との区別を意識します。

**ディクテーションでの着眼点**  
`where` や `when` が聞こえたら、疑問ではなく「前の名詞の説明」かどうかを確認します。直前の名詞を意識できると判別しやすいです。

**入試で問われやすいポイント**  
`where / when / why` を使うのか、`which / that` と前置詞を使うのかを比べる問題が出やすいです。`where = in which`, `when = on which`, `why = for which` の対応で見ると整理しやすいです。`the reason why`、`the day when`、`the way (that)` はまとめて押さえると強いです。

**頻出誤答例**  
`This is the place which I was born.` はそのままでは不自然です。`This is the place where I was born.` または `which I was born in` にします。

**比較メモ**  
後ろの節がほぼ完成していて、足りないのが場所・時・理由・方法なら関係副詞です。人や物そのものを受けるなら関係代名詞を使います。

**例文**

- `This is the place where we first met.` 「ここが私たちが初めて会った場所です。」

**関連カテゴリ**

- 関係代名詞
- 間接疑問
- 名詞節

## 2. 不定詞・動名詞・分詞

### 不定詞

**ひとことで言うと**  
不定詞は `to + 動詞の原形` で、「すること」「するための」「するために」など広く働く形です。

**形の目印**  
`want to do`, `decide to do`, `to learn English` のように、名詞・形容詞・副詞の働きをします。

**コアイメージ**  
まだ実現していないこと、これから向かうこと、目的として置かれることが多い形です。まずは「これから先向き」の感覚を持つと分かりやすいです。

**よくあるつまずき**  
動名詞との区別が難しいですが、動詞ごとに後ろに `to do` を取るか `-ing` を取るかが決まりやすいので、かたまりで覚えるのが有効です。特に `remember`, `forget`, `try` は、後ろの形で意味が変わります。

**ディクテーションでの着眼点**  
`to` が聞こえたら、前置詞なのか不定詞なのかをすぐ判断します。直後に動詞の原形が来るなら、不定詞の可能性が高いです。

**入試で問われやすいポイント**  
不定詞は、名詞的用法・形容詞的用法・副詞的用法の見分けと、`to` の後ろの形がよく問われます。動詞ごとに `to do` を取るかどうか、`疑問詞 + to do` へ言い換えられるかも空所補充で頻出です。

**頻出誤答例**  
`I want going to the library after school.` は誤りです。`want` はふつう `to do` を取るので、`I want to go ...` にします。

**比較メモ**  
不定詞は「これから・目的・予定」と相性がよく、動名詞は「経験済み・事実・習慣」と結び付きやすいです。迷ったら、前の動詞がどちらを要求するかを先に見ます。

**例文**

- `She decided to go to the library after school.` 「彼女は放課後に図書館へ行くことに決めました。」

**関連カテゴリ**

- 動名詞
- 疑問詞＋to不定詞
- 目的構文

### 疑問詞＋to不定詞

**ひとことで言うと**  
疑問詞＋to不定詞は、「何をすべきか」「どうやってするか」を一つのかたまりで表す形です。

**形の目印**  
`what to do`, `how to speak`, `which to choose`, `where to go` のように、疑問詞の後ろに `to + 動詞の原形` が続きます。

**コアイメージ**  
疑問の内容を、そのまま名詞のようなまとまりにした形です。完全な文ではなく、「方法」「選択」「内容」をコンパクトに言えます。

**よくあるつまずき**  
間接疑問と似ていますが、疑問詞＋to不定詞では、後ろに主語がないことが多く、文ではなく句として働きます。

**ディクテーションでの着眼点**  
疑問詞の直後に `to` が聞こえたら、この形を疑います。`how to`, `what to`, `which to` は頻出のまとまりです。

**入試で問われやすいポイント**  
`疑問詞 + to do` と `疑問詞 + 主語 + 動詞` の区別はよく問われます。後ろに主語があるかどうかを見れば、間接疑問との違いを判定しやすくなります。

**頻出誤答例**  
`I don't know what should do next.` は不自然です。`what to do next` か、節にするなら `what I should do next` にします。

**比較メモ**  
後ろに主語がなく句として働くなら `疑問詞 + to do` です。後ろに主語があって平叙文語順なら間接疑問を考えます。

**例文**

- `I want to learn how to speak English fluently.` 「英語を流暢に話す方法を学びたいです。」

**関連カテゴリ**

- 不定詞
- 間接疑問
- 名詞節

### 完了不定詞

**ひとことで言うと**  
完了不定詞は、主節より前に起きたことを不定詞で表す形です。

**形の目印**  
`to have + 過去分詞` になります。`seem to have done`, `be glad to have done` などの形でよく使われます。

**コアイメージ**  
普通の不定詞よりも、「その内容はもう先に起きている」という時間差を出します。主節の時点より前だと考えると整理しやすいです。

**よくあるつまずき**  
`to do` との違いが見えにくいですが、`to have done` なら「すでにそうした」という意味が出ます。

**ディクテーションでの着眼点**  
`to have` が聞こえたら、完了不定詞を疑います。特に `seems to have ...`, `is said to have ...` はまとまりで覚えると再現しやすいです。

**入試で問われやすいポイント**  
`to do` と `to have done` の選択、`seem / appear / be said to have done` の言い換えは出やすいです。主節より前に起きたことかどうかを時間関係で判断します。

**頻出誤答例**  
`He seems finished the work.` は誤りです。主節より前に終わっている内容なので、`He seems to have finished the work.` が自然です。

**比較メモ**  
`to have + 過去分詞` が見えたら完了不定詞です。今につながるなら現在完了、過去の一時点より前なら過去完了を考えます。

**例文**

- `He is said to have been a famous actor.` 「彼は有名な俳優だったと言われています。」

**関連カテゴリ**

- 不定詞
- 現在完了
- 過去完了

### 動名詞

**ひとことで言うと**  
動名詞は、`-ing` の形で「〜すること」を表し、名詞のように使う形です。

**形の目印**  
`enjoy doing`, `finish doing`, `keep doing` など、特定の動詞の後ろによく来ます。

**コアイメージ**  
動作を一つの「もの」「こと」として扱うイメージです。実際に経験したこと、習慣的なこと、一般的なこととも相性がよいです。

**よくあるつまずき**  
不定詞と形が違うだけでなく、意味が変わる動詞があります。`remember doing` と `remember to do` の違いは代表例です。

**ディクテーションでの着眼点**  
動詞の直後に `-ing` が来たら、進行形とは限りません。前の動詞が `enjoy`, `keep`, `stop` などなら、動名詞の可能性を考えます。

**入試で問われやすいポイント**  
`to do` と `doing` の選択は最重要ポイントの一つです。`remember`, `stop`, `try` のように意味差まで絡む動詞は、単なる形の暗記でなく意味まで押さえる必要があります。

**頻出誤答例**  
`I enjoy to listen to music.` は誤りです。`enjoy` の後ろは動名詞なので、`I enjoy listening to music.` にします。

**比較メモ**  
動名詞は「すでに行うもの・一般的に行うもの」をまとまりとして扱いやすい形です。これからする予定や意図を前に出したいなら、不定詞を先に疑います。

**例文**

- `I enjoy listening to music in my free time.` 「私は自由時間に音楽を聴くのを楽しみます。」

**関連カテゴリ**

- 不定詞
- 現在分詞
- 慣用表現

### 現在分詞

**ひとことで言うと**  
現在分詞は、`-ing` の形で名詞を後ろから説明し、「〜している」という能動的な感じを出します。

**形の目印**  
`the boy playing soccer`, `students studying in the library` のように、名詞の後ろに置かれることが多いです。

**コアイメージ**  
その名詞が今している動作、または能動的な性質を補足しています。単なる `-ing` ではなく、「その名詞がする側」だと見ると分かりやすいです。

**よくあるつまずき**  
動名詞や分詞構文と形が同じなので、どの語を説明しているかを確認する必要があります。

**ディクテーションでの着眼点**  
名詞の直後に `-ing` の形が聞こえたら、その名詞の説明かもしれません。文の主動詞だと思い込まないことが重要です。

**入試で問われやすいポイント**  
現在分詞は、進行形と区別して「名詞修飾」だと見抜けるかがよく問われます。`the boy playing soccer` のように、どの名詞を修飾しているかを問う問題も定番です。

**頻出誤答例**  
`The boy plays soccer is my brother.` は語順が崩れています。名詞を後ろから説明するなら `The boy playing soccer ...` または `The boy who plays soccer ...` にします。

**比較メモ**  
名詞が「する側」で、その名詞を説明しているなら現在分詞です。文全体にかかれば分詞構文、名詞のように使えば動名詞を疑います。

**例文**

- `The boy playing soccer is my brother.` 「サッカーをしている少年は私の弟です。」

**関連カテゴリ**

- 動名詞
- 分詞構文
- 過去分詞

### 過去分詞

**ひとことで言うと**  
過去分詞は、名詞を説明して「〜された」「〜してある」のような受け身や完了の感じを出す形です。

**形の目印**  
`the broken window`, `the homework given by the teacher` のように、名詞の前後で修飾に使われます。

**コアイメージ**  
その名詞が「される側」「終わった状態」にあるという感覚です。能動の現在分詞と対比して考えると理解しやすいです。

**よくあるつまずき**  
受動態の文そのものと混同しやすいですが、ここでは文の主動詞ではなく、名詞説明として使われている点が大切です。

**ディクテーションでの着眼点**  
名詞の直後に過去分詞が聞こえたら、そこで一度切らず、名詞説明の続きとして聞きます。`given`, `written`, `spoken` などは頻出です。

**入試で問われやすいポイント**  
現在分詞との対比で、名詞が「する側」か「される側」かを問う問題が多いです。`boring / bored` のような形容詞化した分詞も含め、受け身の感覚を持てるかが重要です。

**頻出誤答例**  
`The homework giving by the teacher was difficult.` は誤りです。宿題は「与える側」ではなく「与えられる側」なので、`given` を使います。

**比較メモ**  
名詞が「される側」なら過去分詞修飾です。文の主動詞として `be + 過去分詞` なら受動態、文全体にかかれば分詞構文です。

**例文**

- `The homework given by the teacher was difficult.` 「先生に出された宿題は難しかったです。」

**関連カテゴリ**

- 現在分詞
- 受動態
- 分詞構文

### 分詞構文

**ひとことで言うと**  
分詞構文は、主節に対して「時」「理由」「条件」「付帯状況」などを、分詞でまとめて言う形です。

**形の目印**  
文頭や文中に `Walking ...`, `Having finished ...`, `Being ...` のようなかたまりが置かれます。

**コアイメージ**  
二つの文を短く圧縮した形だと考えると理解しやすいです。主語はふつう主節と同じで、省略されています。

**よくあるつまずき**  
現在分詞の名詞修飾と混同しやすいですが、分詞構文は文全体にかかる副詞的な働きをします。`Having + 過去分詞` は完了を表します。

**ディクテーションでの着眼点**  
文頭の `-ing` や `Having` が聞こえたら、すぐに主節の動詞を探します。コンマの前後で役割が分かれることが多いです。

**入試で問われやすいポイント**  
分詞構文は、主節との意味関係を問う問題と、`Having + 過去分詞` の時間差を問う問題が出やすいです。主語が主節と一致しているかどうかも、誤文訂正で狙われます。

**頻出誤答例**  
`Walking along the river and I saw a beautiful sunset.` は不自然です。分詞構文なら `Walking along the river, I saw ...` のようにコンマでつなぎます。

**比較メモ**  
コンマを伴って文全体にかかるなら分詞構文です。直前の名詞を説明するだけなら現在分詞・過去分詞の修飾を考えます。

**例文**

- `Walking along the river, I saw a beautiful sunset.` 「川沿いを歩いていると、美しい夕日を見ました。」

**関連カテゴリ**

- 現在分詞
- 過去分詞
- 仮定法過去完了

### enough構文

**ひとことで言うと**  
enough構文は、「〜するのに十分に...」という足りている感覚を表す形です。

**形の目印**  
`形容詞 / 副詞 + enough + to do`、または `enough + 名詞` の形が基本です。

**コアイメージ**  
基準を満たしている、必要量に達しているという意味です。否定感はなく、「できる側」に向かう形です。

**よくあるつまずき**  
`enough` の位置が日本語とずれやすいです。`big enough`, `fast enough` のように、形容詞や副詞の後ろに置くことが多いです。

**ディクテーションでの着眼点**  
`enough` が聞こえたら、その前が形容詞・副詞か、後ろが名詞かを意識します。`to do` まで続いて意味が完成することも多いです。

**入試で問われやすいポイント**  
`enough` の位置は頻出です。`big enough to do` のように後ろへ置くのか、`enough money` のように前へ置くのかを区別できるかが問われます。

**頻出誤答例**  
`He is enough tall to reach the shelf.` は誤りです。形容詞を修飾するなら `tall enough` の順にします。

**比較メモ**  
`enough` は「足りている」側です。「行きすぎてできない」なら `too ... to ...`、単なる二者比較なら比較表現を考えます。

**例文**

- `This box is big enough to hold all the books.` 「この箱はすべての本を入れるのに十分な大きさです。」

**関連カテゴリ**

- too〜to構文
- 不定詞
- 比較

### too〜to構文

**ひとことで言うと**  
too〜to構文は、「あまりに〜なので...できない」という行きすぎを表す形です。

**形の目印**  
`too + 形容詞 / 副詞 + to do` の形になります。

**コアイメージ**  
程度が高すぎて、その後ろの行動ができないというマイナス方向の意味を持ちます。日本語の否定が見えなくても、意味には否定が入ります。

**よくあるつまずき**  
`very` と混同しやすいですが、`too` は「度が過ぎている」感じです。結果的に `cannot` に近い意味になることが多いです。

**ディクテーションでの着眼点**  
`too` が聞こえたら、後ろに `to do` が続くかを待ちます。`too ... to ...` を一つの枠として取ると語順を再現しやすいです。

**入試で問われやすいポイント**  
`too ... to ...` を `so ... that ... not` や `not ... enough to ...` に言い換える問題が出やすいです。意味に否定が含まれることを忘れないのが重要です。

**頻出誤答例**  
`He is too tired that walk any farther.` は誤りです。`too` の後ろは `to do` が基本なので、`too tired to walk ...` にします。

**比較メモ**  
`too ... to ...` は意味に否定を含みます。「十分に〜してできる」なら enough 構文、結果だけを強く述べるなら `so ... that ...` も候補です。

**例文**

- `He was too tired to walk any further.` 「彼は疲れすぎてこれ以上歩けませんでした。」

**関連カテゴリ**

- enough構文
- 結果構文
- 不定詞

## 3. 特殊構文

### 形式主語

**ひとことで言うと**  
形式主語は、文頭の `It` を仮の主語にして、本当の内容を後ろの節に置く形です。

**形の目印**  
`It seems that ...`, `It appears that ...`, `It turned out that ...` のように、文頭の `It` の後ろに評価や判断を表す動詞が来ます。

**コアイメージ**  
先に `It` で文の形を作り、後ろで本当の内容を言います。主語が長くなりすぎないようにする英語らしい並べ方です。

**よくあるつまずき**  
`It that構文` や `It to構文` と近いですが、ここでは主に `seem`, `appear`, `turn out`, `happen` などの動詞が入るタイプを意識すると整理しやすいです。

**ディクテーションでの着眼点**  
文頭の `It` に意味を載せすぎず、後ろの `that節` が本体だと考えます。`It seems that ...` は頻出の聞き取りパターンです。

**入試で問われやすいポイント**  
`It seems that ...` と `He seems to ...` のような言い換えは読解でも文法問題でもよく出ます。`It` 自体を主役だと思わず、後ろに本当の内容があると見抜けるかが重要です。

**頻出誤答例**  
`That he is busy seems.` は不自然です。学校文法で基本形として押さえるなら、`It seems that he is busy.` が自然です。

**比較メモ**  
文頭の `It` の後ろに本体があるなら形式主語です。`It is + 形容詞 + to do` なら It to構文、動詞の後ろの `it` なら形式目的語です。

**例文**

- `It seems that he is busy today.` 「彼は今日忙しいようです。」

**関連カテゴリ**

- It to構文
- It that構文
- 名詞節

### It to構文

**ひとことで言うと**  
It to構文は、`It is + 形容詞 + to do` で、「〜することは...だ」と評価を述べる形です。

**形の目印**  
`It is important to ...`, `It is difficult to ...`, `It is hard to ...` が基本です。

**コアイメージ**  
本当の主語は `to do` の部分ですが、英語ではそれを後ろに置いて、先に評価語を出すことが多いです。

**よくあるつまずき**  
日本語では「〜することは大切だ」と訳すので `To do is important` と考えたくなりますが、英語では `It` を先に置く形が自然です。

**ディクテーションでの着眼点**  
`It is important / difficult / easy ...` が聞こえたら、後ろに `to do` が来ると予測します。評価語を先に押さえると文全体が安定します。

**入試で問われやすいポイント**  
`It is + 形容詞 + to do` の語順は整序問題で崩されやすいです。`to do` を本当の主語と見抜けるか、`of + 人` を伴う形にできるかも問われます。

**頻出誤答例**  
`It is important finish your homework before dinner.` は誤りです。`important` の後ろには `to finish` が必要です。

**比較メモ**  
`It is + 形容詞 + to do` と見えたら It to構文です。後ろが `that節` なら It that構文、動詞の後ろの `it` なら形式目的語です。

**例文**

- `It is important to finish your homework before dinner.` 「夕食前に宿題を終えることは重要です。」

**関連カテゴリ**

- 形式主語
- 不定詞
- 形式目的語

### It that構文

**ひとことで言うと**  
ここでの It that構文は、`It is / was ... that ...` の形で、後ろの `that節` に本当の内容を置くタイプを指します。

**形の目印**  
`It is true that ...`, `It was surprising that ...`, `It is necessary that ...` のような形が代表です。

**コアイメージ**  
先に「その内容をどう評価するか」を述べ、後ろの `that節` で何について言っているかを示します。英語では重い内容を後ろに置く傾向があります。

**よくあるつまずき**  
学校で習う「強調構文」の `It is ... that ...` と名前が似ていますが、ここでは主に「that節を後ろに置く型」として扱っています。そこを混同しないことが大切です。

**ディクテーションでの着眼点**  
`It is true that`, `It was surprising that` のように、評価語の後に `that` が聞こえたら、その後ろが一まとまりの内容になります。

**入試で問われやすいポイント**  
`that` の有無と、後ろが節になることを見抜く問題が出やすいです。`It is important that S 原形 ...` のように、仮定法現在と接続して出題されることもあります。

**頻出誤答例**  
`It is necessary you submit the report today.` は学校文法では不十分です。`that` を入れて `It is necessary that you submit the report today.` にします。

**比較メモ**  
後ろが `that節` で内容説明になるなら It that構文です。後ろが `to do` なら It to構文で、提案や必要を表すときは仮定法現在ともつながります。

**例文**

- `It is necessary that you submit the report today.` 「今日レポートを提出することが必要です。」

**関連カテゴリ**

- 形式主語
- 名詞節
- 仮定法現在

### 形式目的語

**ひとことで言うと**  
形式目的語は、`it` を仮の目的語にして、本当の内容を後ろへ回す形です。

**形の目印**  
`find it easy to do`, `make it clear that ...`, `consider it important to ...` のような形がよく出ます。

**コアイメージ**  
目的語が長いときに、先に `it` を置いて文を軽くし、後ろで本当の内容を説明します。

**よくあるつまずき**  
`it` を普通の目的語だと思うと意味が取りにくくなります。実際に大事なのは、後ろの `to do` や `that節` のほうです。

**ディクテーションでの着眼点**  
`find it easy`, `make it clear` のような表現が聞こえたら、`it` の後ろに評価語、その後ろに本体が来ると考えます。

**入試で問われやすいポイント**  
`it` の直後に何が来るかが並べ替えで狙われます。`it` を本当の目的語だと思わず、後ろの `to do` や `that節` が内容の中心だと判断できるかが重要です。

**頻出誤答例**  
`I found easy to solve this problem.` は誤りです。`easy` の前に仮の目的語 `it` を置いて、`I found it easy to solve ...` にします。

**比較メモ**  
`動詞 + it + 形容詞 + to do / that節` なら形式目的語です。文頭の `It` なら形式主語系で、`it` を普通の目的語だと思わないことが大切です。

**例文**

- `I found it easy to solve this problem.` 「この問題を解くのは簡単だとわかりました。」

**関連カテゴリ**

- It to構文
- It that構文
- 不定詞

### 存在構文

**ひとことで言うと**  
存在構文は、「〜がいる」「〜がある」と新しい情報を導入する形です。

**形の目印**  
`There is ...`, `There are ...`, `There was ...` の形が基本です。後ろに本当の主語が来ます。

**コアイメージ**  
大事なのは `There` ではなく、その後ろに出てくる名詞です。英語ではまず「存在する」という枠を置いてから、本題を出します。

**よくあるつまずき**  
`There` を場所の副詞だと考えないことが大切です。動詞の一致は、後ろの名詞に合わせます。

**ディクテーションでの着眼点**  
`There is` が聞こえたら、その後ろの名詞を待ちます。単数か複数かを意識すると、`is / are` の聞き分けにも役立ちます。

**入試で問われやすいポイント**  
存在構文は、`There is / are` の一致と、後ろに置かれる本当の主語を見抜く問題が出ます。場所の `there` と混同しないことも基本です。

**頻出誤答例**  
`There have many students in the library.` は誤りです。存在構文は `There are many students ...` のように `be` 動詞を使います。

**比較メモ**  
`There is / are` の `there` は場所ではなく枠です。場所を表す副詞の `there` と違い、後ろの名詞が本当の主語になります。

**例文**

- `There is a student waiting for you.` 「あなたを待っている生徒がいます。」

**関連カテゴリ**

- 形式主語
- 現在分詞
- 関係代名詞

### 倒置

**ひとことで言うと**  
倒置は、強調のためにふつうの語順をひっくり返す形です。

**形の目印**  
`Never have I ...`, `Only then did I ...`, `Hardly had I ...` のように、否定語や限定語が文頭に来ると助動詞が前に出ます。

**コアイメージ**  
疑問文の語順に見えても、意味は平叙文です。強調したい語を先に出した結果として、語順が変わっています。

**よくあるつまずき**  
疑問文と混同しやすいですが、文頭の `Never`, `Only then`, `Hardly` などが合図です。

**ディクテーションでの着眼点**  
文頭に強い副詞が来たら、直後に助動詞や `be` 動詞が続く可能性を考えます。ここを予測できると語順を崩しにくいです。

**入試で問われやすいポイント**  
倒置は整序問題で特に出やすいです。否定語や限定語が文頭にあるかを見て、疑問文語順に近い並びになることを思い出せるかが重要です。

**頻出誤答例**  
`Never I have seen such a beautiful place.` は誤りです。倒置なので `Never have I seen ...` にします。

**比較メモ**  
文頭の否定語・限定語のせいで疑問文語順になっているだけで、意味は平叙文です。普通の疑問文と違い、疑問を表したいわけではありません。

**例文**

- `Never have I seen such a beautiful place.` 「こんなに美しい場所は見たことがありません。」

**関連カテゴリ**

- 比較
- 接続詞
- 助動詞

### 使役動詞

**ひとことで言うと**  
使役動詞は、「人に〜させる」「〜してもらう」を表す動詞です。

**形の目印**  
`make + 人 + 動詞の原形`, `have + 人 + 動詞の原形`, `have + 物 + 過去分詞` が代表です。

**コアイメージ**  
主語が直接行動するのではなく、誰かに動作を起こさせる感覚です。`have my hair cut` のように、依頼してしてもらう形も重要です。

**よくあるつまずき**  
`make` の後ろは原形、受け身になると `be made to do` になる点が特に要注意です。

**ディクテーションでの着眼点**  
`made me`, `had my hair`, `kept me` などが聞こえたら、その後ろの形が原形か過去分詞かを意識します。

**入試で問われやすいポイント**  
`make / have / let` の後ろの形、`have + 物 + 過去分詞`、`be made to do` の受け身は定番です。どの型かを見分けるだけで選択肢をかなり絞れます。

**頻出誤答例**  
`She made me to clean the classroom.` は誤りです。`make + 人 + 原形` が基本なので、`made me clean` にします。

**比較メモ**  
使役動詞は「させる・してもらう」型です。知覚動詞は「見た・聞いた内容」を表すので、同じ原形でも意味の軸が違います。

**例文**

- `She made me clean the classroom.` 「彼女は私に教室を掃除させました。」

**関連カテゴリ**

- 知覚動詞
- 受動態
- 形式目的語

### 知覚動詞

**ひとことで言うと**  
知覚動詞は、「見た」「聞いた」「感じた」内容を、目的語の後ろに続けて表す形です。

**形の目印**  
`see / hear / feel / watch / notice + 人 / 物 + 原形 / -ing` が基本です。

**コアイメージ**  
原形なら動作全体、`-ing` なら進行中の場面をとらえることが多いです。映像で切り取る感覚を持つと分かりやすいです。

**よくあるつまずき**  
使役動詞と形が似ていますが、こちらは「させる」ではなく「知覚した内容」を言っています。

**ディクテーションでの着眼点**  
`saw him`, `heard someone` のあとに来る語形が重要です。原形か `-ing` かで意味の細かい差が出ます。

**入試で問われやすいポイント**  
原形と `-ing` の違いは頻出です。動作全体を見るのか、進行中の場面を見るのかを区別できるかが問われます。使役動詞との識別も重要です。

**頻出誤答例**  
`I saw him to cross the street.` は誤りです。知覚動詞の後ろは `to` なしで、`saw him cross ...` または `saw him crossing ...` にします。

**比較メモ**  
知覚動詞は「見た内容」を続ける型です。人に行動を起こさせるなら使役動詞、名詞を説明する `-ing` なら現在分詞を疑います。

**例文**

- `I saw him crossing the street.` 「私は彼が通りを渡っているのを見ました。」

**関連カテゴリ**

- 使役動詞
- 現在分詞
- 動名詞

### 受動態

**ひとことで言うと**  
受動態は、「〜される」「〜された」と、動作を受ける側を主語にする形です。

**形の目印**  
`be + 過去分詞` が基本です。必要なら `by ...` で動作主を添えます。

**コアイメージ**  
誰がしたかより、何がどうされたかを前に出したいときに使います。英語では情報の焦点を変えるためによく用いられます。

**よくあるつまずき**  
過去分詞の名詞修飾と混同しやすいですが、受動態ではそれが文の主動詞になっています。`be made to do` のような形も重要です。`by ...` は必ず必要なわけではなく、だれがしたかが大事なときだけ添えることも押さえます。

**ディクテーションでの着眼点**  
`is spoken`, `was repaired`, `must be finished` のように、`be` と過去分詞のセットをまとまりで取ると安定します。

**入試で問われやすいポイント**  
能動態と受動態の書き換え、助動詞を含む受動態、使役動詞の受け身は特に出やすいです。`be + 過去分詞` を見たら、主語が動作をする側か受ける側かを確認します。助動詞があれば `must be finished` のように `助動詞 + be + 過去分詞` になる形も定番です。

**頻出誤答例**  
`This book reads by many students.` は誤りです。受動態なら `This book is read by many students.` のように `be + 過去分詞` にします。

**比較メモ**  
主語が動作をする側なら能動態、受ける側なら受動態です。`be + 過去分詞` が文の主動詞なら、まず受動態を疑います。

**例文**

- `English is spoken all over the world.` 「英語は世界中で話されています。」

**関連カテゴリ**

- 過去分詞
- 使役動詞
- 助動詞

## 4. 時制・法・助動詞

### 助動詞

**ひとことで言うと**  
助動詞は、義務・可能・推量・意志など、話し手の気持ちを動詞に加える語です。

**形の目印**  
`must`, `should`, `can`, `will`, `may` などの後ろは、ふつう動詞の原形になります。

**コアイメージ**  
単なる意味訳より、「どれくらい強く言っているか」を意識すると理解しやすいです。`must` は強い義務、`should` は助言、`can` は可能が基本です。

**よくあるつまずき**  
日本語訳だけで覚えると使い分けが曖昧になります。文脈の強さを意識することが大切です。

**ディクテーションでの着眼点**  
助動詞が聞こえたら、その直後は原形だと予測します。短い語なので落としやすく、空欄でも狙われやすい部分です。

**入試で問われやすいポイント**  
助動詞は意味の強さの違いと、後ろが原形になる点が頻出です。`must`, `should`, `can`, `may` の基本用法に加え、受動態や完了形と結び付いた形も確認しておくと安定します。

**頻出誤答例**  
`He can speaks English very well.` は誤りです。助動詞の後ろは原形なので、`can speak` にします。

**比較メモ**  
助動詞の後ろは原形です。完了の `have + 過去分詞` や受動態の `be + 過去分詞` と組み合わさっても、まず助動詞の意味と後ろの原形を押さえます。

**例文**

- `You must finish this report by tomorrow.` 「明日までにこのレポートを終えなければなりません。」

**関連カテゴリ**

- 仮定法
- 倒置
- 受動態

### 現在完了

**ひとことで言うと**  
現在完了は、過去の出来事が今につながっていることを表す形です。

**形の目印**  
`have / has + 過去分詞` になります。`for`, `since`, `just`, `already`, `yet`, `ever`, `never` がよく一緒に出ます。`for` は期間、`since` は始点を表すのが基本です。

**コアイメージ**  
「今までの経験」「今も続いている状態」「今ちょうど終わったこと」「結果が今残っていること」の4つが中心です。どれも、過去の出来事を「今まで」や「今」に結び付けて見る感覚が共通しています。

**よくあるつまずき**  
単純過去と違って、`yesterday` のようなはっきりした過去時点とは相性がよくありません。今とのつながりがあるかで考えると分かりやすいです。

**ディクテーションでの着眼点**  
`have` は弱く発音されやすいので聞き落としやすいです。`just`, `already`, `yet` などの副詞も現在完了の合図になります。

**入試で問われやすいポイント**  
現在完了は、単純過去との選択問題が最頻出です。`for / since / already / yet / ever / never` があるか、文末に明確な過去時点がないかをまず確認します。`today`, `this week` のようにまだ終わっていない時間表現は現在完了と相性がよい点も押さえます。

**頻出誤答例**  
`I have seen him yesterday.` は誤りです。`yesterday` があるので、`I saw him yesterday.` が基本です。

**比較メモ**  
現在完了は「今まで」や「今への影響」を見る形です。`yesterday`, `last year`, `when I was a child` のように終わった過去時点が見えたら、まず過去形を考えます。

**例文**

- `He has lived here for ten years.` 「彼はここに10年間住んでいます。」

**関連カテゴリ**

- 過去完了
- 完了不定詞
- 助動詞

### 過去完了

**ひとことで言うと**  
過去完了は、過去のある時点よりさらに前に起きたことを表す形です。

**形の目印**  
`had + 過去分詞` になります。文中に、基準になるもう一つの過去があることが多いです。

**コアイメージ**  
過去を二段階で考える形です。「そのときには、もう〜していた」という感覚を持つとつかみやすいです。

**よくあるつまずき**  
単純過去と見分けにくいですが、文の中に「あとに起きた過去」があるかを探すと整理しやすくなります。

**ディクテーションでの着眼点**  
`had` は短く弱く発音されることがあるので注意します。`by the time`, `when`, `before` などの時間表現と一緒に出やすいです。

**入試で問われやすいポイント**  
過去完了は、基準になる過去が文中にあるかを見抜く問題が多いです。仮定法過去完了と違って、`would have` がなく、事実の時間関係を述べている点が区別の決め手です。

**頻出誤答例**  
`When I arrived at the station, the train already left.` は不自然です。到着より前に出発していたので、`had already left` にします。

**比較メモ**  
基準が過去の一時点なら過去完了です。今につながるなら現在完了、反事実なら仮定法過去完了を疑います。

**例文**

- `By the time I arrived, the meeting had already started.` 「私が到着した時には、会議はすでに始まっていました。」

**関連カテゴリ**

- 現在完了
- 仮定法過去完了
- 完了不定詞

### 仮定法

**ひとことで言うと**  
仮定法は、事実そのものではなく、「現実とずれた想像」「願望」「後悔」を表す考え方です。

**形の目印**  
`wish`, `if only`, `as if`, `would`, `could`, `had` などがよく出ます。時制は「いつの事実とずれているか」を示します。

**コアイメージ**  
一番大切なのは「現実ではない」という感覚です。過去形や過去完了形は、必ずしも時間だけでなく、現実との距離も表しています。

**よくあるつまずき**  
普通の条件文と混同しやすいですが、仮定法では「実際にはそうではない」含みが強く出ます。

**ディクテーションでの着眼点**  
`wish`, `if only`, `as if` が聞こえたら、現実と違う話が始まると考えます。`would`, `could`, `had` の組み合わせにも注目します。

**入試で問われやすいポイント**  
仮定法は、条件文と混ぜて出題されることが多いです。`wish`, `if only`, `as if` を見たら、単純な時制問題ではなく「現実とのずれ」を問う問題だと考えると解きやすくなります。

**頻出誤答例**  
`I wish I can speak English better.` は誤りです。今の事実に反する願望なので、`could speak` にするのが基本です。

**比較メモ**  
仮定法は「現実とのずれ」が中心です。実際に起こりうる条件を述べるだけなら条件文、提案や要求で原形が出るなら仮定法現在です。

**例文**

- `I wish I could speak English better.` 「もっと英語が上手に話せたらいいのに。」

**関連カテゴリ**

- 仮定法過去
- 仮定法過去完了
- 仮定法現在

### 仮定法現在

**ひとことで言うと**  
仮定法現在は、提案・要求・必要性のあとで、動詞の原形を使う形です。

**形の目印**  
`suggest that S do ...`, `It is necessary that S do ...`, `It is important that S be ...` のように、`that節` 内で原形が出ます。

**コアイメージ**  
今の事実ではなく、「そうあるべきだ」「そうしてほしい」という内容を言う形です。時制より、要求や提案の気持ちが中心です。

**よくあるつまずき**  
`he takes` ではなく `he take`、`he is` ではなく `he be` になる点が日本語話者には特に難しいです。

**ディクテーションでの着眼点**  
`suggested that`, `necessary that`, `important that` が聞こえたら、後ろに原形が来る可能性を考えます。三単現の `-s` を付けない点が要注意です。

**入試で問われやすいポイント**  
`that節` 内の動詞が原形になるかどうかは頻出です。`be` がそのまま出る形、三単現の `-s` を付けない形は誤答選択肢の典型です。

**頻出誤答例**  
`The teacher suggested that he goes home early.` は誤りです。仮定法現在なので、`goes` ではなく原形の `go` を使います。

**比較メモ**  
`that節` 内が原形なら仮定法現在を疑います。今の事実に反する想像なら仮定法過去・過去完了で、ここは提案・要求の型です。

**例文**

- `He suggested that we take a short break.` 「彼は私たちに短い休憩を取るよう提案しました。」

**関連カテゴリ**

- It that構文
- 助動詞
- 仮定法

### 仮定法過去

**ひとことで言うと**  
仮定法過去は、今の事実に反する仮定を表す形です。

**形の目印**  
`if + 過去形`, `主節 + would / could + 動詞の原形` が基本です。`be` 動詞では `were` がよく使われます。

**コアイメージ**  
時間としては現在の話でも、形は過去形になります。これは「現実から一歩離れた想像」を示すためです。

**よくあるつまずき**  
条件文と似ていますが、仮定法過去では、現実にはそうではない含みがあります。`If I were you` は定型として押さえたい表現です。

**ディクテーションでの着眼点**  
`if` の後ろが過去形で、主節に `would` や `could` が聞こえたら仮定法過去の可能性が高いです。`were` は聞き取りの重要ポイントです。

**入試で問われやすいポイント**  
`if節 = 過去形`, `主節 = would / could + 原形` の対応は頻出です。特に `If I were you`、`If it were not for ...` のような定型表現はそのまま問われやすいです。

**頻出誤答例**  
`If I were rich, I will travel around the world.` は誤りです。仮定法過去なので、主節は `would travel` にします。

**比較メモ**  
`if + 過去形, would / could + 原形` なら仮定法過去です。未来の現実的な条件なら条件文、過去の反事実なら仮定法過去完了を考えます。

**例文**

- `If I were you, I would study harder.` 「もし私があなたなら、もっと一生懸命勉強するでしょう。」

**関連カテゴリ**

- 条件文
- 仮定法
- 仮定法過去完了

### 仮定法過去完了

**ひとことで言うと**  
仮定法過去完了は、過去の事実に反する仮定を表す形です。

**形の目印**  
`if + had + 過去分詞`, `主節 + would have + 過去分詞` が基本です。

**コアイメージ**  
「あのとき〜していたら、...だったのに」という後悔や振り返りの感覚です。過去について、実際とは違う世界を言っています。

**よくあるつまずき**  
過去完了と形が似ていますが、こちらは事実の説明ではなく、反事実の想像です。`would have` があるかどうかを見ると区別しやすいです。

**ディクテーションでの着眼点**  
`if` の後ろで `had`、主節で `would have` が聞こえたらこの形を疑います。助動詞と完了形の組み合わせをまとめて取ることが大切です。

**入試で問われやすいポイント**  
`if節 = had + 過去分詞`, `主節 = would have + 過去分詞` の対応は整序問題と空所補充の定番です。過去完了との違いは、「事実の説明」か「反事実の想像」かで判断します。

**頻出誤答例**  
`If I had known the truth, I would tell you.` は誤りです。過去の反事実なので、`would have told you` にします。

**比較メモ**  
`if + had + 過去分詞, would have + 過去分詞` なら過去の反事実です。事実の前後関係を述べるだけなら過去完了を使います。

**例文**

- `If she had studied more, she would have passed the test.` 「もっと勉強していれば、彼女は試験に合格していたでしょう。」

**関連カテゴリ**

- 過去完了
- 仮定法過去
- 仮定法

## 5. 比較・強調・慣用

### 比較

**ひとことで言うと**  
比較は、2つ以上のものを比べて、差・同等・比例変化を表す形です。

**形の目印**  
`-er than`, `more ... than`, `as ... as`, `the more ..., the more ...` が基本です。

**コアイメージ**  
何と何を、どの点で比べているかが中心です。形容詞や副詞の形だけでなく、比較の軸を意識すると意味が取りやすくなります。

**よくあるつまずき**  
`more interesting` と `interestinger` のような形の使い分け、`as ... as` の両側をそろえること、`the more ..., the more ...` の対応関係が難所です。

**ディクテーションでの着眼点**  
`than`, `as`, `more`, `-er` が合図になります。前半で比較が始まったら、後半に比較対象が来ると予測します。

**入試で問われやすいポイント**  
比較級・原級・比例表現の使い分けは頻出です。`than` があるのに原級を使っていないか、`as ... as` の左右が対応しているかをまず確認します。

**頻出誤答例**  
`He is more taller than his brother.` は誤りです。比較級を重ねず、`taller than` にします。

**比較メモ**  
`than` や `as ... as` で二者以上を比べるのが比較です。範囲の中で一番なら最上級、程度が高すぎる・十分なら `too` や `enough` を考えます。

**例文**

- `This problem is easier than I expected.` 「この問題は思ったより簡単です。」

**関連カテゴリ**

- 最上級
- 結果構文
- enough構文

### 最上級

**ひとことで言うと**  
最上級は、ある範囲の中で「いちばん〜だ」と言う形です。

**形の目印**  
`the + 最上級`, `the most + 形容詞`, `in / of ...` がセットで出やすいです。

**コアイメージ**  
ただ「すごい」と言うのではなく、比べる範囲の中で頂点に置く形です。範囲表現まで含めて一まとまりで見るのが大切です。

**よくあるつまずき**  
`the` を落としやすい点と、比較級との使い分けに注意します。`by far the best` のような強調もよく出ます。

**ディクテーションでの着眼点**  
`the best`, `the tallest`, `the most difficult` が聞こえたら、その後ろに `in his class`, `of all ...` などの範囲表現が来るかを待ちます。

**入試で問われやすいポイント**  
`the` の有無、`-est` と `the most` の使い分け、範囲表現との組み合わせがよく問われます。比較級との混同を防ぐには、「範囲の中で一番か」を確認すると有効です。

**頻出誤答例**  
`She is tallest student in her class.` は誤りです。最上級はふつう `the` を伴うので、`the tallest student` にします。

**比較メモ**  
最上級は範囲の中の頂点を示します。二者比較なら比較級、程度の強さだけなら結果構文や強調表現と区別します。

**例文**

- `She is the tallest student in her class.` 「彼女はクラスで一番背が高いです。」

**関連カテゴリ**

- 比較
- 慣用表現
- 倒置

### 慣用表現

**ひとことで言うと**  
慣用表現は、単語ごとの意味よりも、まとまりで覚えたほうが自然に使える定型表現です。

**形の目印**  
`be used to ...`, `would rather ... than ...`, `had better do`, `cannot help doing`, `take care of`, `make up one's mind` など、決まった組み合わせで出ます。

**コアイメージ**  
一語ずつ訳すより、チャンクとして覚えるのが本質です。特に前置詞や動詞の組み合わせは、学校文法の知識だけでは再現しにくい部分です。

**よくあるつまずき**  
`used to do` と `be used to doing` の違い、`had better` の意味、`cannot help doing` のような不自然に見える直訳しにくい形がよく混乱ポイントになります。

**ディクテーションでの着眼点**  
既知のかたまりとして音で覚えると強いカテゴリです。短い機能語を落としやすいので、`to`, `of`, `than`, `better` などを含めてまとめて聞きます。

**入試で問われやすいポイント**  
慣用表現は、前置詞や動詞の組み合わせを問う空所補充で頻出です。`used to`, `be used to`, `had better`, `cannot help` のような形は、丸ごと覚えているかがそのまま得点差になります。

**頻出誤答例**  
`I am used to get up early.` は誤りです。`be used to` の `to` は前置詞なので、`getting up` にします。

**比較メモ**  
一語ずつではなく、形ごと覚えるのが基本です。`used to`, `be used to`, `had better`, `cannot help` のように、似た見た目でも後ろの形が変わる点を比べます。

**例文**

- `She is used to getting up early.` 「彼女は早起きすることに慣れています。」

**関連カテゴリ**

- 動名詞
- 不定詞
- 助動詞

## まず優先して復習したい項目

- 関係代名詞 / 関係副詞
  - 長文読解、並べ替え、空所補充のどれでも出やすいです。
- 不定詞 / 動名詞 / 分詞
  - `to do`, `doing`, `done` の区別は設問化されやすいです。
- 現在完了 / 過去完了 / 仮定法
  - 時制と助動詞の組み合わせで差が付きやすい分野です。
- 形式主語 / It to構文 / It that構文 / 形式目的語
  - `It` が何を指しているかを見抜けると文構造が安定します。
- 条件文 / 結果構文 / 目的構文 / 譲歩表現
  - 接続の意味を取り違えると、内容理解と文法問題の両方で失点しやすいです。
