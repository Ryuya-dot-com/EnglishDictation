# 文法復習セクション設計メモ

## 目的

- `questions.json` の各カテゴリを、あとから一貫した方針で復習解説に落とし込めるようにする
- 日本の英語学習者、特に中学生後半から高校生、受験者の自習を想定する
- 文法を「暗記事項」ではなく「聞き取り・語順再現に効く見方」として整理する

## このドキュメントで先に固定すること

- 解説の対象読者
- Web検索で参照する出典の優先順位
- 1項目ごとの説明テンプレート
- カテゴリ同士が重なるときの扱い
- 将来 `文法の復習` セクションを実装するときの章立て

## 想定読者

- 学校文法に一度触れたが、名前と中身が結びついていない学習者
- リスニングやディクテーションで「意味は何となく分かるが、語順を再現できない」学習者
- 参考書の説明が抽象的すぎると感じる学習者

## 学習者向けの説明方針

### 1. まず形より先に「何を言いたい文か」を示す

例:

- `仮定法過去`: 今の事実とは違う想像
- `現在完了`: 過去の出来事が今につながっている感覚
- `形式主語`: 文頭の `It` は席取りで、本当の内容は後ろ

### 2. 次に「見分ける目印」を短く示す

- 目で見て分かる形
- 聞こえたときに拾うべき語
- 似た項目との違い

### 3. 最後に「このアプリでどう役立つか」へ戻す

- どの語順を落としやすいか
- 何を先に予測すると聞き取りやすいか
- 穴埋めでどの語形が狙われやすいか

## 軸をぶらさない執筆ルール

### 用語

- 用語は日本の学校文法で一般的な言い方を優先する
- 難しい用語を使うときは、直後にやさしい言い換えを置く
- 同じ意味の用語は統一する

統一例:

- `if節` / `主節`
- `先行詞`
- `仮の主語` / `本当の主語`
- `動詞の原形`
- `過去分詞`

### 文体

- です・ます調
- 断定しすぎず、学習上の実用を優先する
- 例外は必要なときだけ書く
- 1項目は短く、読んだ直後に問題へ戻れる密度にする

### 1項目の分量目安

- 本文は 250 から 450 字程度
- 例文は 2 から 4 個まで
- 関連カテゴリは 2 から 3 個まで
- 受験者向けに、`どこが設問になりやすいか` を 1 から 2 文で入れる

### 使わない方がよい書き方

- 文法用語だけを並べる説明
- 英文を丸ごと和訳するだけの説明
- 「こう覚えましょう」だけで、見分け方がない説明
- 日本語訳だけでは区別できない項目の説明
- 学習者向け本文に、出典説明や制作メモを混ぜる書き方
- `このアプリでは` `このデータでは` のような内向きの注釈を多用する書き方

## Web検索と出典の方針

### 優先順位

1. 公的・教育機関
2. 英語教育の専門機関
3. 日本の学習者向けに整理された教育サイト

### 今回の基準ソース

- 文部科学省
  - 学習の中心を、知識の暗記だけでなく、`知識及び技能` と `言語活動` を結びつけて設計する方針の確認に使う
- British Council LearnEnglish
  - 文法説明を `clear and simple` にし、例文と練習を結びつける方針の確認に使う
- Benesse の高校英語学習ページ
  - 日本の学習者向けに、文法の見分け方を段階的に説明する書き方の参考に使う

### 出典の使い分け

- 定義と基本の形:
  - British Council を先に見る
- 日本の学習者がつまずく点:
  - Benesse など日本語の学習解説を確認する
- 難易度感と学校文法との整合:
  - 文部科学省の方針と現在のカテゴリ設計を優先する

### 出典メモ

- 文部科学省は、外国語科で `知識及び技能` の習得と、目的や場面に応じた `言語活動` の設計を重視している
- British Council は、文法説明を `clear and simple` にし、例文と理解確認を組み合わせている
- Benesse の説明は、基本から入り、判別ポイントを順に示す構成が多い

### 今回の追加検索で固定する説明ルール

- `条件文 / 仮定法`
  - `if節` は、未来のことでも基本は現在形で置く
  - `if節` が過去形なら、時制の過去ではなく「現実との距離」を表す可能性を先に考える
- `間接疑問`
  - `yes / no` 型は `if / whether`、`wh-` 型は `疑問詞 + 主語 + 動詞` を基本にする
  - `whether` は `or not` を直後に置く形、`to不定詞` の前、前置詞の後で優先される
- `関係詞`
  - 前に名詞があり、後ろの節の中に欠けている要素があるかを見る
  - 定義用法ではコンマを打たず、目的格なら関係代名詞を省略できることがある
  - 関係詞ですでに役割を果たしているので、後ろで代名詞を重ねない
- `不定詞 / 動名詞`
  - 後ろの形は「前の動詞が何を要求するか」で判断する
  - `remember / forget / try` のように、同じ動詞でも `to do` と `doing` で意味が変わるものを重点扱いにする
- `現在完了`
  - `今まで` や `今に影響が残る` 感覚を軸にし、単純過去と切り分ける
  - `for` は期間、`since` は始点として説明する
  - `yesterday` など終わった過去時点とは結び付けない
- `受動態`
  - 基本形は `be + 過去分詞`、助動詞があるときは `助動詞 + be + 過去分詞`
  - `by ...` は必要なときだけ添える情報として扱い、主語が「受ける側」かを優先して見る

## 受験者向けに特に強める観点

### 優先して入れる情報

- 空所補充で狙われやすい語形
- 並べ替えで崩れやすい語順
- 類題で混同しやすい文法項目
- 直後にチェックできる最小限の例文
- 解説直後に確認できる 4 択の `1問チェック`
- 頻出の誤答パターンと、その直し方
- 似た文法を横並びで見分ける比較解説
- 各カテゴリで最短の見分け方を示す `比較メモ`

### 削る情報

- 出典の紹介
- 執筆方針の説明
- 実装上のカテゴリ事情を長く説明する文章
- 初学者向けに細かすぎる例外列挙
- 単独では役立ちにくい細かな豆知識

## 1項目ごとの固定テンプレート

今後の各項目は、次の順で書く。

```md
## {文法カテゴリ名}

### ひとことで言うと
- この文法が何を表すかを一文で説明する

### 形の目印
- まず何を見つければよいか
- 聞き取りで何が合図になるか

### コアイメージ
- 直訳より、意味の中心を説明する

### よくあるつまずき
- 日本語訳だけだと区別しにくい点
- 似たカテゴリとの違い

### ディクテーションでの着眼点
- 落としやすい語
- 語順で注意する点

### 入試で問われやすいポイント
- 空所補充や並べ替えで何が狙われやすいか
- 近い文法とどこで見分けるか

### 頻出誤答例
- 受験者がやりがちな誤答を 1 つ
- どこが違うかを一文で示す

### 比較メモ
- 似た文法と 1 行で区別する
- どの合図を見ればよいかを書く

### 例文
- できれば `questions.json` から 1 から 2 文
- 必要なら補助例文を追加

### 1問チェック
- 各カテゴリにつき 1 問の 4 択にする
- 正解は 1 つだけにする
- そのカテゴリで最も混同しやすい点を問う
- 正解のあとに 1 から 2 文の短い解説を付ける

### 関連カテゴリ
- 混同しやすい項目を 2 から 3 個

### 参考
- 参照したページ名と URL
```

## カテゴリが重なるときの扱い

`questions.json` には、学校文法として近い項目と、アプリ上の学習カテゴリが混在している。そこで、説明を書くときは次のルールで切り分ける。

### `形式主語` と `It to構文` と `It that構文`

- `形式主語` は上位概念として扱う
- `It to構文` と `It that構文` は、学習者が実際に見分ける単位として個別に解説する

### `仮定法` と `仮定法過去` と `仮定法過去完了` と `仮定法現在`

- `仮定法` は総論
- 各下位カテゴリでは、時制と意味の違いに集中する

### `分詞構文` と `現在分詞` と `過去分詞`

- `現在分詞` と `過去分詞` は名詞修飾や形容詞的用法を中心に扱う
- `分詞構文` は副詞的に働くまとまりとして別立てにする

### `接続詞` と `条件文` と `結果構文` と `目的構文` と `譲歩表現`

- `接続詞` は総合カテゴリとして扱う
- 具体的な機能カテゴリは、意味ごとの見分け方を優先する

## 精緻化の方針

説明を厚くするときは、単に文章量を増やすのではなく、次の順で強化する。

1. 誤答しやすい分岐点を明示する
2. 似た文法との比較を入れる
3. 受験での出題形式に結び付ける
4. `1問チェック` で解ける形にする
5. 必要なら最小限の補助例文を足す

長くしても効果が薄い説明より、`どう見分けるか` と `どこで間違えるか` を優先する。
特に `比較メモ` は主要カテゴリだけでなく、全カテゴリにそろえて密度を均一に保つ。

## 今後の作成手順

1. カテゴリごとに `questions.json` の代表例文を 2 から 3 個抜き出す
2. Web で定義、例文、学習者が混同しやすい点を確認する
3. 上のテンプレートに沿って本文を書く
4. 関連カテゴリへのリンクを付ける
5. 用語と文体がこの方針書に合っているか確認する
